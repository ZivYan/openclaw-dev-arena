# 01 - Skill 架构与三重身份

## 核心模型

这个 Skill 的核心不是多 Agent，而是**一个 `coder` + 三种强制身份**：

- **开发者**：实现需求
- **反对者**：专门找错
- **验收者**：只按标准裁决

同一个任务里，这三种身份必须依次出现。

## 总流程

```text
Phase 0 需求锁定
Phase 1 方案对抗
Phase 2 任务拆分
Phase 3 原子开发
Phase 4 自测验证
Phase 5 对抗评审
Phase 6 提交交付
```

## 质量优先级

1. 安全性
2. 正确性
3. 可测试性
4. 一致性
5. 可维护性
6. 开发效率

## 适用范围

- 新功能开发
- Bug 修复
- 测试补充
- 文档同步
- 代码评审
- 提交前收口

## 默认原则

- 先锁需求，再给方案
- 先定义验收，再写实现
- 优先最小改动
- 不做无关重构
- 不忽略错误与验证
- 结论必须能追溯到命令、文件或测试结果
