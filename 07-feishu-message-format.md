# 07 - Phase 5：对抗评审与输出格式

## 目标

把自己当 reviewer 来否定自己的实现。

## 默认执行角色

- 角色：`rd-lead`
- 推荐模型：Claude Opus

## 审查优先级

### P0 安全

- 注入风险
- 敏感信息泄露
- 越权操作

### P1 稳定性

- 空指针 / 未处理异常
- 资源泄漏
- 并发风险
- 错误传播缺失

### P2 正确性

- Happy Path 是否成立
- Error Path 是否覆盖
- 边界条件是否处理
- 文档与实现是否一致

### P3 可维护性

- 是否与现有风格一致
- 是否过度抽象
- 是否留下未来漂移点

## 强制输出

```markdown
## 对抗审查
- 最大风险 1：
- 最大风险 2：
- 最大风险 3：
- 结论：通过 / 退回补改
```

## 回复格式

每次交付默认使用 6 段：

1. 任务定义
2. 方案对比
3. 任务拆分
4. 实现结果
5. 验证结果
6. 对抗审查
