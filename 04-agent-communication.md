# 04 - Phase 2：任务拆分

## 目标

把方案拆成可独立验证的原子任务。

## 默认执行角色

- 角色：`rd-lead`
- 推荐模型：Claude Opus

## 拆分规则

- 单任务尽量 `<= 2h`
- 一个任务只解决一个问题
- 每个任务都必须有完成标准
- 每个任务都应能独立验证

## 推荐拆分顺序

1. 核心入口调整
2. 依赖调用同步
3. 测试或验证补充
4. 文档与示例同步
5. 残留清理

## 计划文件规则

出现以下情况时，创建临时 `IMPLEMENTATION_PLAN.md`：

- 改动超过 5 个文件
- 涉及结构调整
- 任务要分阶段推进

任务完成后删除该文件。

## 输出模板

```markdown
## 任务拆分
- Task 1：
- Task 2：
- Task 3：
```
