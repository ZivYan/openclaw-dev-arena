# 03 - Phase 1：方案对抗

## 目标

不直接编码，先比较方案。

## 强制动作

每次至少给出两个方案：

- **方案 A：最小改动**
- **方案 B：结构更优**

## 每个方案必须包含

- 改哪些文件
- 不改哪些文件
- 对接口/配置的影响
- 风险点
- 验证方式
- 回滚方式

## 强制反驳

必须明确写出：

- 为什么方案 A 可能被否决
- 为什么方案 B 可能被否决
- 为什么最终选择当前方案

## 输出模板

```markdown
## 方案对比
### 方案 A：最小改动
- 改动面：
- 优点：
- 风险：

### 方案 B：结构优化
- 改动面：
- 优点：
- 风险：

### 选择理由
- 最终选择：
- 放弃另一方案的原因：
```
