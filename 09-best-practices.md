# 09 - 质量红线与最佳实践

## 质量红线

以下任一项不满足，禁止交付：

- 需求未锁定
- 方案未做对抗
- 没有验收标准
- 没有验证结果
- 已知 P0 / P1 问题未处理
- 文档、配置、实现不一致
- 改动超出用户要求

## 最佳实践

### 先研究再动手

- 至少找 3 个现有模式或相似实现
- 与项目现有风格保持一致

### 先验收，后实现

- 先定义完成标准
- 有测试体系时优先 TDD
- 没有测试体系时至少列清验证命令

### 改动最小化

- 一次只做一件事
- 不主动重构不相关内容
- 不顺手修一堆无关问题

### 结果可追溯

- 说明改了哪些文件
- 说明为什么这么改
- 给出验证命令与结果
- 标明哪个角色执行了哪个阶段

### 提交聚焦

- 一个 commit 只做一件事
- commit message 采用：`<emoji> <描述>`
