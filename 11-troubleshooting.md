# 11 - 失败与受阻升级策略

## 什么时候算受阻

- 同一问题连续失败 3 次
- 无法确定正确方案
- 本地上下文不足以安全决策
- 验证结果与预期冲突

## 受阻后必须输出

```markdown
## 受阻说明
- 已尝试方案：
- 具体错误：
- 失败原因分析：
- 备选方案 1：
- 备选方案 2：
```

## 升级顺序

1. 回到 Phase 0，检查需求是否锁准
2. 回到 Phase 1，检查方案是否选错
3. 缩小任务范围，重新拆分
4. 检查角色路由是否错配（如评审任务误派给 `rd-coder`）
5. 请求用户确认方向

## 常见失误

- 没锁需求就开始写
- 没有验收标准就开始改
- 验证只跑成功路径
- 把“解释”当成“交付”
- 明知有风险还强行提交
